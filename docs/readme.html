SimpleFlight C++

Version 2007.08.08

This package contains the C++ version of SimpleFlight. SimpleFlight is a variable fidelity, modular flight dynamics model that can be run standalone or in conjunction with another simulation. To build SimpleFlight, run “make” from the top level directory. The standalone executable and shared library can both be created

Building SimpleFlight

  1. Change to the top level directory

  2. Run “make”. Use “make lib” to build the shared library or “make exe” to build the executable. “make all” builds both

Running The Standalone Version

SimpleFlight can be run on the command line with the following input parameters:

<Input XML Configuration File> <Number of seconds to execute (sec)> <frame rate in Hz>

Using the Library

SimpleFlight can be run externally by creating a new FDMGlobals object, populating it with modules, and passing the initialization information (in the form of the input XML file) to the FDMGlobals object. The model can be stepped externally by calling update() on the FDMGlobals with a given timestep in seconds. See the main.cpp listing for an example.

Using SimTimer

A simulation timer is provided. Use the start() method to execute in wall-clock time. Use startContstructive() to run at maximum rate.

Building New Modules

New FDM modules can be created and added to an instance of SimpleFlight. Every module subclasses FDMModule. Look at the modules provided for guidance on module creation. If you would like to load the module from an XML input file, you must reference the module in the ModuleLoader class. New modules can access XML-based input data by using the Node and NodeUtil classes. The XML input file can contain additional information not listed in the example file. Simply add new XML entities to be used as input by new Modules.
